Notable Findings
01

Parallel execution in route.ts — Promise.all([fetchRepoTree, getOrBuildSystemMap]) runs the domain and system pipelines simultaneously. Neither waits for the other.

02

55 prefix entries, sorted longest-first. DOMAIN_PREFIXES uses longest-match-wins: src/server/auth matches before src/server. Order in the array is intentional.

03

"Source-confirmed" ≠ semantically understood. Strong evidence means a keyword matched near a filename in the GitHub tree blob list — not that the file was parsed or its logic was analysed.

04

Two different cache TTLs. System map: 60-min in-process. Repo tree: 10-min in-process. Both reset every serverless cold-start — no persistence between deployments.

05

v1 reserved fields are always null. integrityScore, trustBoundaryNote, and weakSeams are typed but never populated — placeholders for v2 import-chain analysis.

06

~70 npm packages mapped to 8 systems. PACKAGE_SIGNALS covers @clerk/nextjs → Auth, prisma → Database, openai → AI Orchestration, stripe → Payments, and more.

07

GitHub token never leaves the server. The API route fetches with the user's decrypted token (AES-256-GCM) server-side. The response JSON contains only tree data — no credentials.
